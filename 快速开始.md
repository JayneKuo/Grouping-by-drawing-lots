## 📝 部署到Vercel

### 步骤1：推送到GitHub

```bash
git add .
git commit -m "更新代码"
git push
```

### 步骤2：在Vercel配置环境变量（重要！）

**重要**：`.env.local` 文件不会提交到Git，所以需要在Vercel中手动配置。

1. **登录** [Vercel Dashboard](https://vercel.com/dashboard)
2. **选择你的项目**（如果没有，先导入GitHub仓库）
3. **进入设置**：
   - 点击项目名称
   - 点击顶部 **Settings**
   - 左侧选择 **Environment Variables**
4. **添加环境变量**：
   
   **变量1：**
   - Key: `VITE_GIST_ID`
   - Value: `2cabed01d5406fa3ea1334f2e116bc90`
   - Environment: 全选（Production、Preview、Development）
   - 点击 **Save**
   
   **变量2：**
   - Key: `VITE_GIST_TOKEN`
   - Value: `你的Token（以ghp_开头）`
   - Environment: 全选（Production、Preview、Development）
   - 点击 **Save**

5. **重新部署**：
   - 回到项目首页
   - 点击 **Deployments** 标签
   - 找到最新部署，点击 **...** → **Redeploy**

### ⚠️ 重要提示

- **Token安全**：Token只在Vercel Dashboard中配置，不要提交到Git
- **必须配置**：如果不配置环境变量，Vercel部署后无法同步数据
- **详细说明**：查看 `Vercel部署说明.md` 获取更详细的步骤

## 🔧 常见问题

### Q: 数据没有同步？

**检查：**
1. 打开浏览器控制台（F12），查看是否有错误
2. 确认 `.env.local` 文件已创建
3. 重启开发服务器

### Q: 无法更新数据？

**检查：**
1. Token是否正确配置
2. Token是否有gist权限
3. 查看控制台错误信息

### Q: 其他用户看不到数据？

**检查：**
1. Gist是否设置为公开（Public）
2. 所有用户是否使用相同的Gist ID
3. 数据是否已成功同步到Gist
