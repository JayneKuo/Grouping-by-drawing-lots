# 数据库配置说明

## 🎯 两种运行模式

### 模式1：简化版（推荐，无需MySQL）✅

**特点：**
- ✅ 无需安装MySQL
- ✅ 无需配置数据库
- ✅ 开箱即用
- ⚠️ 数据保存在内存中，重启后丢失

**启动方式：**
```bash
# 双击运行
快速启动（无需数据库）.bat

# 或命令行
npm start
```

**默认账号：** admin / admin123

---

### 模式2：完整版（需要MySQL）

**特点：**
- ✅ 数据持久化存储
- ✅ 支持多人协作
- ✅ 适合生产环境
- ⚠️ 需要安装和配置MySQL

**安装步骤：**

1. **安装MySQL**
   - 下载：https://dev.mysql.com/downloads/mysql/
   - 安装时记住root密码

2. **自动配置数据库**
   ```bash
   # 双击运行
   自动配置数据库.bat
   ```
   
   或手动配置：
   ```bash
   # 1. 编辑配置文件
   # 打开 backend/.env 文件
   # 填入MySQL密码：DB_PASSWORD=your_password
   
   # 2. 初始化数据库
   cd backend
   npm run init-db
   ```

3. **启动完整版**
   ```bash
   npm run start-full
   ```

---

## 🔍 检查MySQL是否安装

```bash
mysql --version
```

如果显示版本号，说明已安装。

---

## 💡 建议

- **个人使用/测试**：使用简化版（无需MySQL）
- **多人协作/生产环境**：使用完整版（MySQL）

---

## 🆘 常见问题

### Q: MySQL安装失败？
A: 使用简化版，无需MySQL也能运行

### Q: 数据库连接失败？
A: 
1. 检查MySQL服务是否启动：`net start mysql`
2. 检查密码是否正确（backend/.env文件）
3. 或使用简化版

### Q: 简化版数据会丢失？
A: 是的，重启服务器后数据会丢失。如需持久化，请使用MySQL版本。
