# 服务器启动状态

## ✅ 已自动启动服务器

服务器已在后台启动，请等待几秒钟让服务器完全启动。

## 📋 检查服务器状态

### 方法1：查看浏览器
1. 打开浏览器访问：http://localhost:8080 （或 http://localhost:8081）
2. 如果页面能打开，说明前端服务器运行正常

### 方法2：测试后端API
在浏览器访问：http://localhost:3000/api/health

应该返回：
```json
{"status":"ok","message":"服务器运行正常（简化版，无需MySQL）"}
```

### 方法3：查看控制台
如果你有命令行窗口打开，应该能看到：
- 后端：`🚀 服务器运行在 http://localhost:3000`
- 前端：`VITE v5.x.x  ready in xxx ms`

## 🧪 测试添加选手功能

1. 访问前端地址（http://localhost:8080 或 8081）
2. 登录（admin / admin123）
3. 进入比赛详情页
4. 点击"参赛选手"右侧的"+"按钮
5. 尝试添加选手
6. 查看后端控制台，应该看到"📥 [单个添加]"日志

## ⚠️ 如果还是404

请检查：
1. 后端控制台是否显示"已注册的路由"列表
2. 浏览器Network标签中的请求URL是否正确
3. 后端控制台是否有任何错误信息

## 🔄 如果需要重启

如果服务器没有正常启动，可以：
1. 关闭所有命令行窗口
2. 双击运行：`一次性解决所有问题.bat`
3. 或者手动运行：`npm start`
