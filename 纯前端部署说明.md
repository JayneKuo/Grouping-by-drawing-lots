# 纯前端版本 - 部署说明

## ✅ 完全满足你的需求

- ✅ **纯前端应用** - 无需后端服务器
- ✅ **数据存储在浏览器** - 使用localStorage
- ✅ **支持导出/导入** - 可以保存数据文件
- ✅ **直接部署Vercel** - 只部署前端，无需配置后端
- ✅ **支持增删改查** - 所有操作都在浏览器中完成

## 🎯 工作原理

### 数据存储
- 所有数据存储在浏览器的**localStorage**中
- 支持**导出**为JSON文件保存
- 支持**导入**JSON文件恢复数据
- 支持**多用户同步**（通过导出/导入文件）

### 增删改查
- ✅ **增**：创建比赛、添加选手
- ✅ **删**：删除比赛、删除选手
- ✅ **改**：修改比赛信息、更新比分
- ✅ **查**：查看比赛列表、查看比赛详情

**所有操作都在浏览器中完成，无需后端服务器！**

## 📋 使用方式

### 1. 创建比赛
1. 登录系统（使用内置账号）
2. 点击"新建"按钮
3. 填写比赛信息
4. 创建成功

### 2. 添加选手
1. 进入比赛详情页
2. 点击"参赛选手"右侧的"+"按钮
3. 单个添加或批量添加（每行一个姓名）
4. 添加成功

### 3. 导出数据（备份/分享）
1. 点击底部导航"我的"
2. 点击"数据管理"
3. 点击"导出数据"
4. 下载JSON文件保存

### 4. 导入数据（恢复/同步）
1. 点击底部导航"我的"
2. 点击"数据管理"
3. 点击"导入数据"
4. 选择之前导出的JSON文件
5. 数据自动恢复

### 5. 多用户同步
- **方式1**：用户A导出数据 → 发送给用户B → 用户B导入数据 → 继续操作 → 导出数据 → 发送给用户A → 用户A导入数据
- **方式2**：所有用户使用同一台设备/浏览器（数据自动共享）

## ⚠️ 注意事项

1. **数据存储位置**：
   - 数据存储在浏览器的localStorage中
   - 清除浏览器数据会丢失数据
   - 建议定期导出数据备份

2. **多设备同步**：
   - 需要通过导出/导入文件手动同步
   - 或者使用同一台设备/浏览器

3. **数据备份**：
   - 建议每次重要操作后导出数据
   - 可以将JSON文件保存到网盘

4. **存储限制**：
   - localStorage通常有5-10MB限制
   - 对于小型比赛（<100场比赛）完全够用
